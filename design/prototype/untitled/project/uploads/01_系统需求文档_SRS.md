## 1. 引言

### 1.1 背景与目标
构建一个面向全球的星座性格分析网站，融合三件事：可读性强的星座百科、一套自研的"心理学风格"性格测试，以及一套带个人化补充因子的情感匹配。目标是做出比满大街运势站更可信、更具传播力的产品。

### 1.2 产品定位与免责声明（强约束）
- 产品定位为**娱乐与自我探索工具**，不是科学诊断，也不是心理医疗工具。
- 星座性格描述来自**市场共识**（多家热门占星站点聚合），不代表科学结论。
- 性格测试维度借鉴心理学（近似大五人格），但**不等同于任何专业量表**，不得宣称临床效力。
- **每一处测试入口与每一个结果页都必须展示一行免责声明**（中英双语）。

### 1.3 目标受众与语言
- 全球受众；首发**简体中文 + 英文**双语，界面可一键切换。
- **无需注册登录**：降低门槛、规避 PII 与跨境数据合规风险。

### 1.4 范围
**本期包含**：星座百科、性格测试、年龄因子、异变检测、情感匹配（基础+性格补充+理想型）、结果分享与增长、中英国际化、LLM 增强**接口预留**。

**本期不包含（明确排除）**：用户账户/登录体系、服务端数据库与结果持久化、基于出生时间的真实星盘合盘、LLM 文案的真实调用（仅留接口与降级实现）、原生 App、支付与会员。

### 1.5 术语
| 术语 | 含义 |
|---|---|
| 元素能量画像 | 用户在火/土/风/水四象 + 两条极性轴上的连续打分向量 |
| 共识基线 | 某星座由市场共识描述提炼出的"典型人格向量" |
| 异变 | 用户实测人格与其星座基线偏离超过阈值的状态 |
| 补充因子 | 在传统星座配对之上叠加的、由答题人格计算出的修正项 |
| 理想型 | 用户对期望对象的人格描述，用于提高匹配精度 |

## 2. 总体描述

### 2.1 用户角色
- **访客**：浏览星座百科，不一定测试。
- **测试者**：完成性格测试，获取人格画像与异变结论。
- **匹配求询者**：在测试后发起情感匹配，可选填理想型。
- **分享者**：把结果卡片传播到社交平台（核心增长来源）。
- **内容维护者**：通过配置文件（JSON）维护题库、文案、权重，无需改业务代码。

### 2.2 核心用户故事
1. 作为访客，我想快速读懂某个星座的性格优缺点（中/英），以便了解自己或他人。
2. 作为测试者，我想用几分钟答题，得到不被星座框死、带数据依据的人格画像。
3. 作为年长用户，我希望结论体现"我已超越星座模板"，而不是被一句话概括。
4. 作为匹配求询者，我想知道哪些星座与我更合，并理解"为什么合"。
5. 作为分享者，我想一键生成好看到想截图的结果卡片并分享。
6. 作为英文用户，我希望整站、测试题、结果文案都有自然的英文版本。

### 2.3 假设与约束
- 全部核心计算在**浏览器端**完成，无需后端。
- 结果通过 **URL 编码**实现可分享、可复现，不落库。
- 星座基线、题库、权重均为**版本化的静态数据**，可在不发版业务逻辑的情况下调整。

## 3. 功能需求（FR）

> 优先级：P0 = 必须，P1 = 重要，P2 = 可延后。

### FR-1 星座百科（P0）
- FR-1.1 提供 12 星座详情页，含日期范围、四象/三态属性、守护星、性格优点/缺点、爱情/事业倾向、与各星座的简要相性。
- FR-1.2 内容由 2–3 家热门来源聚合成"共识版"描述，并标注为娱乐性质。
- FR-1.3 全部内容**中英双语**，且对搜索引擎友好（每个星座独立可索引页面）。
- FR-1.4 详情页提供"测一测我是不是典型 XX 座"入口，导流到性格测试。

### FR-2 性格测试问卷（P0）
- FR-2.1 提供一套自研问卷（默认约 20 题，支持快测/完整两档）。
- FR-2.2 题型以情境化单选/二选一为主，避免直白的"你是否外向"，降低自我美化偏差。
- FR-2.3 每题对人格维度加权，作答完成后产出**元素能量画像**（连续打分，非二元类型）。
- FR-2.4 输出为"能量画像 + 原创原型名 + 性格关键词"，**不得使用四字母代码或 MBTI 同名轴**，以与 MBTI 区分。
- FR-2.5 答题流程为分步推进（逐题或小批量），含进度指示，可返回上一题、可重测。
- FR-2.6 结果页须展示驱动结论的**具体维度得分**（透明化，建立可信度）。

### FR-3 年龄因子（P0）
- FR-3.1 测试开始或结束前采集**年龄段**（不采集精确生日，降低 PII）。
- FR-3.2 年龄越大，星座基线对结论的影响越弱、个体化解读越强（详见 AR-2）。
- FR-3.3 结果文案语气随年龄段切换：年轻段偏"星座印证"，年长段偏"经历塑造个体"。

### FR-4 异变检测（P0）
- FR-4.1 比较用户实测人格与其星座的**年龄调整后基线**的偏离度。
- FR-4.2 偏离超过阈值即判定为"异变"，并分档（微异/显异/极异）给出不同强度的展示。
- FR-4.3 异变时，**抛开星座、用用户真实主导维度**重新描述，并附带激励性文案（"这一切都是你一路经历所得——你的人格不再由星座定义，而由你走过的路定义"）。
- FR-4.4 异变率需经校准落在合理区间（约 15%–30%），避免"人人异变"或"无人异变"。
- FR-4.5 异变文案中英双语，且年长用户更易触发（与 FR-3 一致）。

### FR-5 情感匹配（P0）
- FR-5.1 仅在用户完成性格测试后可发起匹配（依赖其人格向量）。
- FR-5.2 用户可选择是否进行匹配（不强制）。
- FR-5.3 匹配综合三部分：传统星座元素相性（基础层）、答题人格补充因子、可选理想型因子（详见 AR-4）。
- FR-5.4 **理想型为可选输入**：用简短选项描述期望对象的人格；填写后匹配结果精度更高、推荐更收窄。
- FR-5.5 输出**排序后的若干推荐星座**，每条附"为什么合"的解释（元素和谐 / 哪些维度相似或互补 / 与理想型契合度）。
- FR-5.6 若用户本人为异变，匹配自动降低传统元素权重、提高人格因子权重（详见 AR-4）。

### FR-6 结果分享与增长（P0）
- FR-6.1 测试结果与匹配结果均可生成**可分享图片卡片**（视觉上"想截图"，含能量罗盘、原型名、关键词、品牌水印）。
- FR-6.2 卡片在**客户端生成**（无需服务器），支持下载与系统级分享（Web Share API，移动端优先）。
- FR-6.3 结果状态编码进 **URL**，分享链接可复现同一结果页（中/英随链接保留）。
- FR-6.4 提供"测一测你和 TA"的对比/引流入口，鼓励二次传播。
- FR-6.5 社交分享元信息（OG/Twitter Card）：本期使用精心设计的**静态默认图**；按结果动态生成的 OG 图列入未来项（依赖边缘函数）。

### FR-7 国际化（中英切换）（P0）
- FR-7.1 全站 UI、星座百科、题库、结果与匹配文案、免责声明均提供中英双语。
- FR-7.2 提供显著的语言切换；首次访问按浏览器语言智能预选，可手动覆盖并记忆偏好（本地存储，非 PII）。
- FR-7.3 中英为**独立可索引路由**，并正确声明 hreflang，利于全球 SEO。
- FR-7.4 文案需为**自然母语表达**而非机翻；保留各自语言的语气风格（异变激励语在两种语言中都要"燃"）。

### FR-8 LLM 增强接口（仅预留，本期不实现）（P1）
- FR-8.1 性格/异变/匹配文案的生成需通过一个**统一的文案 Provider 接口**产出。
- FR-8.2 本期提供**模板/规则库实现**作为唯一启用实现，完全离线、免费、纯静态。
- FR-8.3 预留 **LLM 实现的占位（stub）**：未来可由边缘函数调用大模型现场生成个性化文案，密钥藏于服务端，并带缓存与失败降级到模板实现。
- FR-8.4 接口切换不得改动业务调用方代码（依赖注入/工厂）。详见《技术设计文档》第 8 节。

## 4. 算法需求（行为层，AR）

> 仅定义输入/输出/规则。具体公式、参数与伪代码见《技术设计文档》第 5 节。

### AR-1 人格向量模型
- **输入**：测试各题作答。
- **输出**：六维人格向量——火/土/风/水四象能量 + 内显↔外显、守序↔混沌两条极性轴，各维归一化。
- **规则**：每题对维度加权累加后归一；由主导维与缺失维映射出关键词与原型名。

### AR-2 年龄加权
- **输入**：用户星座基线向量、年龄段。
- **输出**：年龄调整后的"有效基线"。
- **规则**：年龄越大，有效基线越向"中性个体画像"靠拢（星座信号衰减）；同时异变阈值随年龄下降。

### AR-3 异变判定
- **输入**：用户人格向量、年龄调整后基线、年龄相关阈值。
- **输出**：是否异变 + 异变等级 + 触发该判定的主导维度。
- **规则**：偏离度 > 阈值 → 异变；偏离度分段映射等级；输出需可解释（指明是哪些维度造成偏离）。

### AR-4 匹配评分
- **输入**：用户星座、用户人格向量、各候选星座典型画像、（可选）理想型向量、用户异变状态。
- **输出**：候选星座的排序得分 + 每条解释。
- **规则**：综合"元素基础相性 + 人格补充因子（相似度与互补度的加权）+ 理想型契合度（仅在提供时启用）"；用户异变时调整权重以弱化星座、强化人格。

## 5. 数据需求

| 数据集 | 内容 | 维护方式 |
|---|---|---|
| 星座共识基线 | 12 星座属性、双语描述、六维基线向量 | 版本化 JSON / 内容文件 |
| 题库 | 题目、选项、各维度权重（双语） | 版本化 JSON，可配置题量档位 |
| 权重配置 | 年龄段→α、异变阈值参数、匹配权重 | 单独配置文件，便于调参 |
| 匹配矩阵 | 12×12 元素基础相性表 + 每星座微调 | 版本化数据 |
| 文案库 | 关键词、原型名、异变激励语、匹配解释模板（双语） | i18n 资源，供模板 Provider 使用 |

- **数据治理**：内容/文案/权重与业务逻辑**分离**，非工程人员可改文案与权重；所有数据集带版本号，便于灰度与回溯。

## 6. 非功能需求（NFR）

- **NFR-性能**：内容页静态直出，首屏快；测试与匹配为轻量本地运算，结果生成体感即时。目标 Lighthouse Performance ≥ 90。
- **NFR-响应式**：移动优先，覆盖手机/平板/桌面；测试流程在小屏单手可完成。
- **NFR-国际化**：中英文本外置，无硬编码字符串；日期/数字本地化；预留再扩语言能力。
- **NFR-隐私与合规（全球）**：不收集 PII、不强制 Cookie；如启用分析需用隐私友好方案并提供同意横幅（满足 GDPR/CCPA 的最低披露）；提供隐私说明页（双语）。
- **NFR-SEO**：每个星座页与落地页具备语义化结构、元信息、结构化数据与 hreflang。
- **NFR-可访问性**：满足键盘可达、可见焦点、对比度、`prefers-reduced-motion`（WCAG AA 方向）。
- **NFR-安全**：纯静态无后端攻击面；未来边缘函数须隐藏密钥、限流、设缓存。
- **NFR-可维护性**：核心算法为纯函数、可单测；权重/文案配置驱动。
- **NFR-可观测性**：保留接入隐私友好分析（如自托管方案）的能力，用于观察转化与传播漏斗。

## 7. 验收标准（整体）
1. 12 星座中英详情页均可独立访问并被索引，含免责声明。
2. 完成约 20 题测试后，得到六维能量画像、原型名、关键词，并展示维度得分。
3. 同一组答案在不同年龄段下，异变倾向与文案语气有可观察差异。
4. 异变判定可解释，整体异变率落在校准区间。
5. 匹配输出排序推荐与逐条解释；填写理想型后结果收窄且更精准；异变用户权重自动调整。
6. 测试与匹配结果均可生成可分享卡片并通过分享链接复现。
7. 全站可中英一键切换，文案为自然母语表达。
8. 文案均经由 Provider 接口产出，替换为 LLM 实现时调用方代码零改动。

## 8. 附录：与传统占星的关系
本产品对"合盘/星盘"仅取**太阳星座 + 元素相性**这一最易懂、最普及的层面作为基础层，并在其上叠加自研人格因子。基于出生时间的完整星盘（月亮、金星、火星、上升等相位）属于更高阶玩法，列为未来扩展，不在本期范围。
